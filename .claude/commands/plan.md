# Plan Skill

You are in PLAN-ONLY mode. Do NOT write or edit any code.

1. Read the user's request carefully
2. Break it into numbered steps
3. For each step, describe:
   - WHAT you will change (specific functions, classes, constants)
   - WHERE (exact file paths)
   - WHY (what problem it solves or what it adds)
   - RISK: any layout, overflow, or backward-compat concerns
4. If the change touches UI, describe the visual design (sizes, colors, positions, layout)
5. If the plan has 3+ parts, number them as separate phases with clear boundaries
6. Present the full plan and STOP

Do NOT write any code until the user says "approved" or "go ahead".

If requirements are unclear, ask clarifying questions BEFORE presenting the plan.
